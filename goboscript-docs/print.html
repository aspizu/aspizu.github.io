Automatic Installation

goboscript comes with a bash script which tries to automatically install all dependencies and install goboscript without user intervention.

Open a terminal window and type these commands.

Open some directory where you would like to store goboscript.

cd $HOME/downloads/packages

Clone the goboscript repository.

git clone https://github.com/aspizu/goboscript

Open the repository directory.

cd goboscript

Run the automatic install script using bash.

bash ./install.sh

If everything goes well, goboscript will be installed.

Verify that goboscript is installed and is working.

cd test/
gsc

Manual Installation

If the automatic install fails, continue with these instructions.

goboscript depends on the following dependencies:

  • Python
  • pip
  • setuptools
  • Lark parser

Try to install these dependencies using your operating system's built-in package manager. If any python module is not available, try to install it using pip.

After installing all the dependencies, re-run the automatic install script.

Creating a goboscript project

To create a blank goboscript project, create a directory to hold project files and open it inside a terminal window.

gsc --init

This will create the following files.

my-project/
|- stage.gobo
|- main.gobo
|- blank.svg

Compiling the project

Being inside the root of the project directory, this will compile the project and output a Scratch project with the same name as the directory.

gsc

You can specify a different name for the output file.

gsc -o build.sb3

goboscript project structure

Each sprite gets its own .gobo file. You are free to arrange costumes however you want.

The stage goes in stage.gobo, as such you cannot use stage.gobo for a sprite named "stage".

Costumes

Each sprite must have atleast one costumes declaration. Write paths in strings separated by commas. Costume paths are relative to the project directory.

costumes "blank.svg", "player.png";

Globs can be used to include costumes based on a pattern.

costumes "frames/*.svg";

This will include all files ending with .svg from the frames/ directory sorted alphabetically.

Costume names

Costumes by default will be named their file-names minus the file extension. So, the costume blank.svg will be named blank in the sprite.

As unix file-names cannot include a / character, if a costume file-name includes {{fwslash}} then it will be replaced with a / character. So, the costume char-{{fwslash}}.png will be named char-/ in the sprite.

Costume machines

goboscript comes with powerful tools to generate costumes programmatically.

*machine:ASCII

Include blank.svg with names being ASCII characters from 33 to 127.

costumes "*machine:ASCII";

Precedence of operators

All binary operators are left to right associative.

This means that 

1 + 2 + 3 + 4

becomes

((([1] + [2]) + [3]) + [4])
Operators in order of precedence
and, or
not, in, =, !=, <, >, <=, >=
&, +, -
*, /, %
unary -, ( expression )

Boolean Coercing

Scratch does not allow putting a round block (Value reporter) inside a angled hole (Expects boolean).

goboscript will workaround this.

variable = true;
if variable {}

set [variable v] to [1]
if <not <(variable) = [0]>> then

Variables

To define a variable, first assign to it.

variable = 10;

set [variable v] to [10]

goboscript provides compound-assignment operators.

variable++;
variable += 2;
variable -= 2;
variable *= 2;
variable /= 2;
variable %= 2;
variable &= ".";

change [variable v] by [1]
change [variable v] by [2]
set [variable v] to ((variable) - [2])
set [variable v] to ((variable) * [2])
set [variable v] to ((variable) / [2])
set [variable v] to ((variable) mod [2])
set [variable v] to (join (variable) [.])

To use a variable, refer to it by its name.

say variable;

say (variable)

Global Variables

Variables by default are "For this sprite only". To make a variable "For all sprites", there are two ways.

Either add the variable name in a variables declaration in stage.gobo.

variables global_variable;

or assign to the variable in stage.gobo.

onflag {
  global_variable = 0;
}

Custom Blocks

To define a custom block,

def custom_block arg1, arg2, arg3 {}

To define a custom block with the "Run without screen refresh" unchecked.

nowarp def custom_block arg1, arg2, arg3 {}

define custom_block (arg1) (arg2) (arg3)
...

Arguments

goboscript does not support boolean arguments, use boolean coercing and the true and false constants.

To refer to a custom block's arguments use the $argument syntax.

Using custom blocks

custom_block "one", "two", "three";

custom_block [one] [two] [three]:: custom

Local Variables

goboscript allows for custom block scoped local variables using the local keyword.

def foo {
  local var = 1;
  say var;
}

define foo
  set [foo:var v] to [1]
  say (foo:var)

Warning

Remember that these are just regular "For this sprite only" variables. Using them with recursion will have unexpected results.
